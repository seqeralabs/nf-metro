## v0.5.0 — diamond fork-joins, multi-line labels, multiple file icons

*2026-02-24* · [Release](https://github.com/pinin4fjords/nf-metro/releases/tag/0.5.0)

### Diamond fork-joins

Stations that a line passes straight through - splitting to parallel paths and immediately rejoining - now render as flat diamond bubbles rather than a pair of overlapping pills. This pattern is common at QC branch points where multiple tools run in parallel before merging back into the main route.

![Diamond fork-join stations](../assets/renders/section_diamond.svg)

### Multi-line station labels

Station labels can now span multiple lines using `\n`:

```
node[First line\nSecond line]
```

![Multi-line labels](../assets/renders/multiline_labels.svg)

### Multiple file icons per station

A single station can now carry more than one file icon, for cases where a tool emits multiple distinct outputs:

```
node[Tool]:::file:output.bam,file:output.bai
```

![Multiple file icons](../assets/renders/05b_multi_icons.svg)

### Configurable section spacing

`--x-spacing` and `--y-spacing` flags (and their `%%metro` equivalents) now accept any value above safe minimums, rather than a small fixed set of presets. This makes it easier to tune diagram proportions for different screen sizes or embedding contexts.

### New pipeline examples

Two new real-world pipeline examples were added to the gallery:

**nf-core/epitopeprediction** — an immunoinformatics pipeline predicting epitopes from peptide sequences.

![nf-core/epitopeprediction](../assets/renders/pipeline_epitopeprediction.svg)

**nf-core/hlatyping** — HLA typing from sequencing reads.

![nf-core/hlatyping](../assets/renders/pipeline_hlatyping.svg)

### Other improvements

- Animated balls reduced to 1 per track (was 3), for less visual noise
- Dark-mode CSS fix for transparent-background SVGs
- Terminus stations nudged away from passing lines to prevent overlap
- Fork/join bubbles widened for labels; labels flip away from exit ports
- Bypass gap2 vertical offset avoids overlap with L-shape routes
- Labels centre correctly on non-station text elements
- Exit gap now only applies to single-layer sections
