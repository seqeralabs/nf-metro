## v0.1 — first release

*2026-02-16* · [Release](https://github.com/pinin4fjords/nf-metro/releases/tag/0.1)

The initial public release of nf-metro, establishing the core pipeline: parse a Mermaid `.mmd` file with `%%metro` directives, compute a metro-map layout, render to SVG.

### Auto-inferred section layout

The layout engine infers section positions from the graph topology. For most pipelines, no `%%metro grid:` directives are needed - the engine walks the section dependency graph and places sections automatically.

![Auto-layout: nf-core/rnaseq](../assets/renders/pipeline_rnaseq_auto.svg)

For cases where the inferred layout doesn't match your intent, explicit grid positions override the inference:

```
%%metro grid: preprocessing | 0,0
%%metro grid: alignment     | 1,0
%%metro grid: quantification| 2,0
```

### Animated balls

`nf-metro render mypipeline.mmd --animate` adds travelling balls that follow each metro line from source to destination, giving an intuitive sense of data flow direction.

### File terminus icons

Stations that represent file inputs or outputs can carry a small file icon:

```
node[Output BAM]:::file
```

![File terminus icons](../assets/renders/05_file_icons.svg)

### Transparent background for light theme

`%%metro style: light` produces an SVG with a transparent background, suitable for embedding in light-coloured docs pages without a visible background rectangle.

### CI/CD and PyPI publishing

GitHub Actions workflows for CI (test + lint on pull requests) and publishing to PyPI on release were set up from day one.
