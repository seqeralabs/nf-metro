## v0.4.0 — Nextflow DAG import, bypass routing below sections

*2026-02-19* · [Release](https://github.com/pinin4fjords/nf-metro/releases/tag/0.4.0)

### Nextflow DAG import

nf-metro can now import Mermaid files produced by Nextflow's `--with-dag` flag directly, with no manual editing required. The importer detects unsupported DAG constructs and reports them clearly rather than producing a broken render.

```bash
nf-metro render pipeline_dag.mmd -o pipeline.svg
```

Nextflow DAG files use a flat graph (no subgraphs for sections), so the auto-layout engine places sections automatically.

![Variant calling pipeline from Nextflow DAG import](../assets/renders/nf_variant_calling.svg)

![Nextflow pipeline with subworkflows](../assets/renders/nf_with_subworkflows.svg)

### Bypass routing below intervening sections

Lines that skip over one or more sections now route *below* those sections rather than diagonally through them. This keeps bypasses visually separate from the section content they skip.

![Bypass routing around a section](../assets/renders/around_section_below.svg)

### Span-based line ordering

`%%metro line_order: span` sorts lines in the legend by their total span across the graph - useful when you want long-running routes to appear first.

### Hidden station prefix

Stations prefixed with `_` are hidden during rendering but participate in layout. Useful for routing control points that shouldn't appear as stations on the map.

### Other improvements

- Minimum inter-section gap enforced for smooth bypass corners
- Separate bypass routes sharing the same inter-column gap
- Uniform corner radius for bypass U-shape routes
- Bypass gap1 vertical shifted toward source for section clearance
- Terminus stations nudged away from passing lines
- Vertical centering of section content
- Empty subgraphs containing only edges handled correctly
