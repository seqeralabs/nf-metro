# v0.7.x

[GitHub releases](https://github.com/pinin4fjords/nf-metro/releases/tag/0.7.0) · [Full changelog](https://github.com/pinin4fjords/nf-metro/compare/0.6.1...0.7.2)

The 0.7 family represents the largest single batch of changes since the initial release: 228 commits between 0.6.1 and 0.7.0 alone, covering new output formats, three new directives, richer visual options, and a substantial reliability push via the layout invariant framework.

Existing `.mmd` files render with no changes - everything new here is additive.

## v0.7.2 — rowspan bbox shrink fix

*2026-05-18* · [Release](https://github.com/pinin4fjords/nf-metro/releases/tag/0.7.2)

Patch release fixing a layout regression visible in v0.7.1. Sections that declared `grid_row_span > 1` but whose content fit in one row had their bounding box pinned open by a neighbouring section - blocking the shrink that would have removed the overlap.

Most visible on the [nf-core/differentialabundance](https://github.com/nf-core/differentialabundance) metro map, where the `data_prep` section (rowspan=2) previously ended mid-air between rows.

![nf-core/differentialabundance metro map](https://pinin4fjords.github.io/nf-metro/0.7.2/assets/renders/differentialabundance.svg)

Three gallery fixtures changed (pure bbox shrinks, no station or edge movement); all others byte-identical.

## v0.7.0 — layout invariants, fragility reduction, and new directives

*2026-05-17* · [Release](https://github.com/pinin4fjords/nf-metro/releases/tag/0.7.0)

### Embed your maps anywhere

`nf-metro render mypipeline.mmd --format html` now produces a self-contained, interactive HTML file you can drop straight into a docs page or a blog post.

### Three new `%%metro` directives

**`off_track: node_id`** lifts file inputs above the line tracks so they sit clear of the metro lines instead of breaking them. Great for FASTA/GTF inputs hanging off a workflow.

![Off-track file inputs](https://pinin4fjords.github.io/nf-metro/0.7.2/assets/renders/off_track_convergence.svg)

**`center_ports: true`** (or `--center-ports`) centres inter-section ports on the shorter section, which tidies up many fan-in/fan-out cases.

**`legend_min_height: N`** reserves a minimum legend height so a tall map's legend doesn't look stranded.

### Dashed and dotted line styles

Lines can now be **dashed** or **dotted** to indicate optional or conditional routes through a pipeline. Add a fourth field to your `%%metro line:` directive:

```
%%metro line: qc | Quality Control | #2196F3 | dashed
%%metro line: optional_path | Optional | #ff8800 | dotted
```

Both the route ink and the legend swatch reflect the style:

![Simple pipeline with dashed QC line](https://pinin4fjords.github.io/nf-metro/0.7.2/assets/renders/simple_pipeline.svg)

### Richer file icons

Two new icon types join the existing single `file`:

- **`files`** — stacked pages, for paired-end or multi-file outputs
- **`dir`** — folder icon, for directory outputs

All three accept an optional caption to label exactly what's flowing in:

```
node[Label]:::files:Read 1, Read 2
```

![Multi-file icons](https://pinin4fjords.github.io/nf-metro/0.7.2/assets/renders/05b_multi_icons.svg)

![Files (stacked) icon](https://pinin4fjords.github.io/nf-metro/0.7.2/assets/renders/05c_files_icon.svg)

![Directory icon](https://pinin4fjords.github.io/nf-metro/0.7.2/assets/renders/05d_folder_icon.svg)

### New gallery example: nf-core/differentialabundance

The [nf-core/differentialabundance](https://github.com/nf-core/differentialabundance) pipeline was added to the gallery.

### Numbered sections in reading order

Sections are now numbered in visual reading order (left-to-right, top-to-bottom), so "1, 2, 3..." matches how you scan the map.

### Layout invariant framework

A phase-boundary guard system (`compute_layout(validate=True)`) catches layout engine bugs at the offending phase rather than several phases later. Combined with full-corpus parametrised tests and the C13 row-gap runtime guard, the engine is substantially more robust to new topology patterns.

The layout pipeline is now organised into six named stages with flat `Stage X.Y` naming - much easier to reason about than the old monolithic phase numbering.

### Under the hood

- Fan-in merge junctions route cleanly onto the trunk
- Per-line path grouping gives consistent line z-order where lines cross
- Animated balls no longer fly off-piste at merge junctions
- Cached `station_lines()` across ~40 call sites (O(1) amortised)
- A new [CONTRIBUTING.md](https://github.com/pinin4fjords/nf-metro/blob/main/CONTRIBUTING.md) covering the visual review workflow, invariant ratchet, and git hygiene expectations
